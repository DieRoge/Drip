# Drip Programming Language ☕ 

**Drip** — это легковесный, интерпретируемый язык программирования с «кофейным» синтаксисом, разработанный на языке F# с использованием библиотеки парсер-комбинаторов **FParsec**. 

Язык сочетает в себе императивный подход с элементами функционального программирования, поддерживая замыкания, рекурсию и строгую типизацию.

## Основные характеристики
- **Парадигма:** Процедурно-функциональная.
- **Типизация:** Динамическая, строгая.
- **Платформа:** .NET 8.0+.
- **Синтаксис:** На основе отступов (indentation-based), чувствителен к регистру.

## Быстрый старт

### Требования
- [.NET SDK 8.0](https://dotnet.microsoft.com/download) или выше.

### Установка и запуск
1. Склонируйте репозиторий:
   ```bash
   git clone https://github.com/DieRoge/Drip
   cd Drip
   ```
2. Выполните сборку проекта, используя команду:
```bash
dotnet build
```
3. Запустите вашу программу (файл с расширением .drip):

```Bash
dotnet run path/to/your_script.drip
```

## Руководство по языку
### 1. Типы данных
| Тип     | Литерал в Drip | Примечание                    |
|---------|----------------|-------------------------------|
| Integer | 42, -7         | Целые числа.                  |
| Float   | 32.6, 0.5      | Числа с плавающей точкой.     |
| String  | "Espresso"     | Строки в двойных кавычках.    |
| Boolean | Coffee / Tea   | Coffee == true,Tea == false.  |

### 2. Переменные
Объявление переменной начинается с ключевого слова brew, за которым следует оператор пайпа ```|>```.
```Drip
brew strength |> 10
brew message |> "Your coffee is ready"
```
### 3. Вывод и ввод
pour — выводит значение на экран.

sip — считывает строку из консоли.

```Drip
pour "What is your name?"
brew user_name |> sip
pour "Hello, " + user_name
```
### 4. Управляющие конструкции (Aroma)
Условный оператор выполнен в стиле кофейного аромата:

aroma [условие] hot [ветка true]

iced [ветка false]
```Drip
brew sugar |> 5
aroma sugar == 0 hot
    pour "Black coffee"
iced
    pour "Sweet coffee"
```
### 5. Функции и рекурсия
Функции определяются через brew и ключевое слово order для списка аргументов.

```Drip
brew sum |> order a, b ~>
    a + b

pour sum(5, 10)
```
### 6. Комментирование кода
В языке Drip предусмотрены однострочные комментарии для документирования кода. Всё, что начинается с символа # и продолжается до конца строки, полностью игнорируется интерпретатором.
```Drip
# Это пример однострочного комментария
brew coffee_strength |> 5  # Устанавливаем крепость напитка
pour coffee_strength
```
## Примеры программ
Вычисление числа Фибоначчи (Демонстрация полноты по Тьюрингу)
Drip поддерживает глубокую рекурсию и арифметические операции, что делает его вычислительно полным.

```Drip
# Классическая рекурсивная реализация
brew fib |> order n ~>
    aroma n == 0 hot 0
    iced
        aroma n == 1 hot 1
        iced fib(n - 1) + fib(n - 2)

pour "Fibonacci for 10 is:"
pour fib(10)
```
Калькулятор суммы до N (Цикл через рекурсию)
```Drip
brew sumTo |> order n, acc ~>
    aroma n == 0 hot acc
    iced sumTo(n - 1, acc + n)

pour sumTo(100, 0) # Выведет 5050
```
## Использование генеративного ИИ
Проект разработан с использованием семейства моделей Google Gemini 3 (Flash, Deep Think и 3.1 Pro). ИИ использовался как интеллектуальный ассистент и инструмент для рефакторинга.

Роли моделей:
Gemini 3.1 Pro:
- Проектирование высокоуровневой архитектуры проекта.
- Разработка начальных шаблонов парсера на F# с использованием FParsec.
- Консультации по системе типов AST (Abstract Syntax Tree).

Gemini 3.0 Deep Think:
- Глубокий рефакторинг кода и устранение проблем с «жадным» парсингом вызовов функций.
- Разработка логики обработки замыканий и рекурсивных вызовов в Eval.fs.
- Анализ сложных ошибок рантайма и предложения по архитектуре обработки отступов.

Gemini 3 Flash:
- Быстрая отладка мелких ошибок (Runtime Errors).
- Генерация документации и стилистические правки кода.
- Ответы на синтаксические вопросы по стандартной библиотеке F#.

Доля участия ИИ: Весь ключевой код был проанализирован, модифицирован и протестирован вручную. ИИ использовался для ускорения написания бойлерплейт-кода и как экспертная система для решения проблем компиляции в функциональной парадигме.
